# Story 3.35 — Gestão de Usuários: CRUD Completo

## Status: Done

## Story
Como administrador do sistema, quero gerenciar usuários (listar, criar, editar, excluir) através de uma interface frontend completa, para controlar o acesso à plataforma.

## Acceptance Criteria
- [x] Endpoint GET /usuarios lista todos os usuários
- [x] Endpoint POST /usuarios cria novo usuário com senha SHA-256
- [x] Endpoint PUT /usuarios/{id} atualiza usuário (senha opcional)
- [x] Endpoint DELETE /usuarios/{id} remove usuário
- [x] Página /configuracoes/usuarios exibe tabela com id, usuario, nome, empresa
- [x] Botão "Novo Usuário" abre modal com formulário
- [x] Formulário: usuario (obrigatório), nome, empresa, senha (obrigatória no criar, opcional no editar)
- [x] Botões editar e excluir por linha com confirmação de exclusão
- [x] Busca por nome, login ou empresa
- [x] ruff check 0 erros nos arquivos da story

## Implementation Notes
- Senha: SHA-256 hex digest (64 chars) via `hashlib.sha256`
- Hash feito no `UsuarioRepository._hash_password()`, não no endpoint
- Schemas Pydantic V2 com `model_config = ConfigDict(from_attributes=True)`
- Frontend usa `usuariosApi` de `@/lib/api/usuarios.ts`
- Componentes em `@/components/configuracoes/`: `UsuariosTable`, `UsuarioFormModal`
- `/configuracoes` redireciona para `/configuracoes/usuarios`
