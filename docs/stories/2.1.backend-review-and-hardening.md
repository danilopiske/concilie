## Story

**As a** desenvolvedor/mantenedor do sistema Concilie,
**I want** ter o backend FastAPI revisado e endurecido,
**so that** todos os endpoints estejam seguros, bem testados, com tratamento de erros consistente e prontos para produção.

## Acceptance Criteria

1. **AC1 — Inventário e revisão de endpoints**
   - Todos os endpoints em `apps/api/app/api/v1/endpoints/` estão mapeados
   - Cada endpoint tem: autenticação verificada (`get_current_user`), tratamento de erro com HTTP code correto, resposta tipada com schema Pydantic
   - Endpoints sem autenticação (se houver) estão documentados como intencional

2. **AC2 — Consistência de schemas e models**
   - Todos os models SQLAlchemy em `apps/api/app/models/` têm `__tablename__` definido e estão registrados em `models/__init__.py`
   - Todos os schemas Pydantic têm `model_config = ConfigDict(from_attributes=True)` (Pydantic v2)
   - Não há imports circulares entre models, schemas e routers

3. **AC3 — Tratamento de erros padronizado**
   - Erros de validação retornam HTTP 422 com mensagem descritiva
   - Erros de negócio (not found, duplicate, unauthorized) retornam códigos corretos (404, 409, 401/403)
   - Não há `bare except` ou erros silenciados sem log

4. **AC4 — Cobertura de testes**
   - `pytest apps/api/tests/ -v` passa 100% sem erros
   - Todos os módulos críticos têm ao menos 1 teste de caminho feliz e 1 de erro
   - Fixtures de DB usam SQLite in-memory (sem dependência de MySQL nos testes)

5. **AC5 — Segurança básica**
   - Nenhum segredo (senha, chave) hardcoded no código — tudo via variáveis de ambiente
   - CORS configurado explicitamente (não `allow_origins=["*"]` em produção)
   - Senhas armazenadas com hash (não plain text) — verificar `users` table

6. **AC6 — Qualidade de código**
   - Sem imports não utilizados em arquivos de endpoints
   - Funções com mais de 50 linhas têm comentário de responsabilidade
   - `relatorio_service.py` e `reconciliation_core.py` revisados para complexidade excessiva
