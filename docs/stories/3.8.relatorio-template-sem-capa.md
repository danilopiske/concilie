## Story

**As a** operador gerando relatórios no Concilie,
**I want** escolher entre o modelo "Completo" (com capa e segunda página) e o modelo "Sem capa e 2ª página" no momento de gerar um relatório,
**so that** eu possa gerar relatórios mais diretos para uso interno ou compartilhamento rápido sem as páginas introdutórias.

## Contexto Técnico

- Templates Jinja2 em `relatorios/`:
  - `template_relatorio.html` — template completo (capa + 2ª página + análise)
  - `template_relatorio_mensal.html` — mensal completo
- `gerar_relatorio_html` em `modules/reports.py` linha ~2327: carrega `template_relatorio.html`
- `gerar_relatorio_mensal_html` em `modules/reports.py` linha ~3849: carrega `template_relatorio_mensal.html`
- `RelatorioRequest` schema: `processamento_id`, `calc_tipo`, `tipo_relatorio` — precisa receber `modelo`
- `metadata` dict em `endpoints/relatorios.py` propaga parâmetros para `run_async_report`
- Frontend: `apps/web/src/app/(dashboard)/relatorios/page.tsx` — enviar campo `tipo_relatorio` e `calc_tipo`

## Dev Notes

### Estrutura atual do `template_relatorio.html`

Inspecionar o arquivo para localizar os blocos de capa/segunda página. Procurar por:
- Tags com classes como `page-cover`, `capa`, `primeira-pagina`, `segunda-pagina`
- `page-break-after: always` no CSS que separa a capa das demais seções
- Comentários `<!-- CAPA -->` se existirem

A estratégia é: identificar os blocos, remover do template sem capa, manter o restante intacto.

### Seleção dinâmica do template em `reports.py`

```python
# gerar_relatorio_html — ANTES:
template = env.get_template("template_relatorio.html")

# DEPOIS:
template_name = "template_relatorio_sem_capa.html" if modelo == "sem_capa" else "template_relatorio.html"
template = env.get_template(template_name)
```
