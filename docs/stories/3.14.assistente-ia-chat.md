## Story

**As a** operador do Concilie,
**I want** fazer perguntas em linguagem natural sobre os dados de um processamento (ex: "qual bandeira cobrou mais acima do contratado em março?", "quantas transações de débito tiveram taxa acima de 3%?"),
**so that** eu consiga extrair insights específicos dos dados sem precisar navegar por múltiplas telas ou exportar dados para análise manual.

## Acceptance Criteria

1. **AC1 — Endpoint de chat contextual**
   - `POST /ia/chat`
   - Body:
     ```python
     class ChatRequest(BaseModel):
         mensagem: str
         processamento_id: Optional[str] = None  # contexto específico
         cliente_id: Optional[int] = None
         historico: List[dict] = []  # [{"role": "user"|"assistant", "content": "..."}]
     ```
   - Resposta:
     ```python
     class ChatResponse(BaseModel):
         resposta: str
         dados_contexto: Optional[dict] = None  # dados usados para responder
         sugestoes: List[str] = []              # próximas perguntas sugeridas
     ```
   - Monta contexto com sumário do processamento (totais por bandeira/modalidade, desvios se houver)
   - Envia contexto + pergunta ao LLM com system prompt específico do Concilie
   - Não armazena histórico no servidor — cliente envia histórico a cada request

2. **AC2 — System prompt especializado**
   - System prompt define o domínio: conciliação financeira de cartões, taxas MDR, adquirentes brasileiros
   - Instrui o LLM a responder com base nos dados fornecidos no contexto
   - Instrui a sugerir 2-3 perguntas de follow-up relevantes
   - Idioma: sempre responder em português

3. **AC3 — Contexto dinâmico do processamento**
   - `AIService.montar_contexto(processamento_id)` → string com sumário estruturado:
     ```
     Processamento ID: 123 | Cliente: Nome | Período: Jan/2024
     Total transações: 15.432 | Valor total: R$ 2.3M
     Por bandeira:
       - Visa Crédito: 8.500 tx, taxa média 2.8%, desvio +0.3% vs contratado
       - Mastercard Débito: 4.200 tx, taxa média 1.5%, dentro do contratado
     Períodos ausentes: nenhum
     Abusividade detectada: Sim (3 alertas críticos)
     ```
   - Limita contexto a ~2000 tokens para não exceder limites da API

4. **AC4 — Interface de chat no frontend**
   - Expandir/substituir `apps/web/src/app/(dashboard)/ia/page.tsx`
   - Seletor de processamento no topo (opcional — chat geral sem contexto também funciona)
   - Interface de chat com bolhas de mensagem (user/assistant)
   - Chips de sugestões de perguntas abaixo de cada resposta do assistente
   - Loading state com "digitando..." enquanto aguarda resposta
   - Histórico da conversa mantido em estado local (não persiste no banco)
   - Exemplos de perguntas na tela inicial: "Qual bandeira cobrou mais?", "Houve cobranças acima do contratado?", "Quais períodos tiveram menos transações?"

5. **AC5 — Qualidade e segurança**
   - API key do LLM lida de variável de ambiente (`GEMINI_API_KEY`) — nunca hardcoded
   - Rate limiting: máximo 10 requests/minuto por usuário autenticado
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando (mock da API do LLM nos testes)

### Padrão de chamada Gemini

```python
import google.generativeai as genai

genai.configure(api_key=settings.GEMINI_API_KEY)
model = genai.GenerativeModel(
    model_name=settings.GEMINI_MODEL,
    system_instruction=SYSTEM_PROMPT
)

# Converter histórico para formato Gemini
gemini_history = [
    {"role": msg["role"] if msg["role"] == "user" else "model", "parts": [msg["content"]]}
    for msg in historico
]

chat = model.start_chat(history=gemini_history)
response = chat.send_message(f"Contexto:\n{contexto}\n\nPergunta: {mensagem}")
resposta = response.text
```

### Rate limiting simples (em memória)

```python
from collections import defaultdict
from datetime import datetime, timedelta

_rate_limit: dict = defaultdict(list)

def check_rate_limit(user_id: str, max_per_minute: int = 10) -> bool:
    now = datetime.utcnow()
    window_start = now - timedelta(minutes=1)
    _rate_limit[user_id] = [t for t in _rate_limit[user_id] if t > window_start]
    if len(_rate_limit[user_id]) >= max_per_minute:
        return False
    _rate_limit[user_id].append(now)
    return True
```

### Concerns (dívida técnica)
- **[MEDIUM] Rate limit em memória não funciona em multi-process** — `_rate_limit` é variável de módulo; não persiste entre reinicializações nem compartilha estado em multi-worker. Requer Redis para produção.
- **[MEDIUM] `historico: List[dict]` sem validação de schema** — campos "role" e "content" não são validados; dict malformado resulta em string vazia silenciosamente.
- **[LOW] `datetime.utcnow()` deprecated no Python 3.12+** — usar `datetime.now(timezone.utc)`.
- **[LOW] `AIService.__new__()` nos testes** — bypassa `__init__`; funcional mas pode mascarar regressões de construção.
