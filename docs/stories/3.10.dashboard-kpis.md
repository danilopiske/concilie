## Story

**As a** operador do Concilie,
**I want** uma página de dashboard executivo como tela inicial do sistema com KPIs consolidados (processamentos, valor conciliado, alertas de abusividade, extratos divergentes) e feed de atividade recente,
**so that** eu tenha visibilidade imediata do estado geral do sistema ao acessar, sem precisar navegar por múltiplas telas para entender o que precisa de atenção.

## Acceptance Criteria

1. **AC1 — Endpoint `GET /dashboard/resumo`**
   - Retorna agregados globais (ou filtrado por `?cliente_id=`):
     ```python
     class DashboardResumo(BaseModel):
         total_processamentos: int
         processamentos_mes_atual: int
         valor_total_conciliado: float      # soma vendas_calculos.valor_venda
         alertas_abusividade_pendentes: int # processamentos com variação > 25%
         extratos_divergentes: int          # ExtratoCliente.status == 'divergente'
         extratos_aguardando: int
         relatorios_gerados_mes: int
         ultimo_processamento: Optional[dict]  # {id, cliente, data, status}
     ```
   - Resposta em < 500ms (queries agregadas com índices)

2. **AC2 — Endpoint `GET /dashboard/atividade-recente`**
   - Retorna os últimos 20 eventos ordenados por data desc:
     ```python
     class EventoAtividade(BaseModel):
         tipo: Literal["importacao", "calculo", "relatorio", "abusividade", "extrato"]
         descricao: str          # "Importação concluída para Cliente X"
         cliente_nome: str
         created_at: datetime
         status: str             # "ok" | "alerta" | "erro"
     ```
   - Agrega de: `Import_Task`, `Calculo_Task`, `Relatorio_Task`, `ExtratoCliente`

3. **AC3 — Página `/dashboard` no frontend**
   - Rota: `apps/web/src/app/(dashboard)/dashboard/page.tsx`
   - 4 KPI cards no topo: Processamentos (mês), Valor Conciliado, Alertas Abusividade, Extratos Divergentes
   - Cada card: ícone + número grande + label + variação vs mês anterior (se disponível)
   - Cores: verde (ok), amarelo (atenção), vermelho (crítico) conforme limites definidos
   - Seção "Atividade Recente": lista cronológica dos últimos 20 eventos com badge de tipo e status

4. **AC4 — Dashboard como rota inicial**
   - Ao acessar `/`, redirecionar para `/dashboard` (ou tornar `/dashboard` a rota padrão do layout)
   - Link "Dashboard" adicionado como primeiro item do sidebar com ícone `LayoutDashboard`

5. **AC5 — Qualidade**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando
   - Endpoint `/dashboard/resumo` documentado no Swagger
