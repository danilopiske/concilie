## Story

**As a** operador do Concilie,
**I want** fazer upload de extratos (arquivos de venda/recebível) vinculados ao cadastro do cliente, e que outro operador possa baixar e importar esses arquivos, com o sistema validando automaticamente quais extratos já foram importados e quais ainda estão pendentes,
**so that** toda a equipe tenha visibilidade centralizada sobre o que foi entregue, o que foi processado e se há divergências entre o que foi enviado e o que entrou no sistema.

## Contexto Técnico

- `Cliente` (`clientes.cliente_id`) — modelo existente
- `Processamento` (`processamentos`) — tem `cliente_id` + `nome_arquivo` — base para o cruzamento
- `ImportTask` — já tem FK `cliente_id → clientes.cliente_id`
- Storage atual: filesystem local (padrão do projeto — sem S3/cloud nesta story)
- Nova model necessária: `ExtratoCliente` (`extratos_cliente` table)

## Acceptance Criteria

1. **AC1 — Model `ExtratoCliente` e migration**
   - Nova tabela `extratos_cliente` com campos:
     - `id` (UUID, PK)
     - `cliente_id` (Integer, FK → clientes.cliente_id, NOT NULL)
     - `nome_arquivo` (String 255, NOT NULL) — nome original do arquivo
     - `caminho_arquivo` (String 500) — path local para download
     - `tipo` (String 50) — `'Venda'` | `'Recebivel'` | `'Outro'`
     - `uploaded_by` (String 100) — usuário que fez upload
     - `uploaded_at` (DateTime, default=now)
     - `status` (String 30) — `'aguardando'` | `'importado'` | `'divergente'`
     - `processamento_id` (Integer, nullable) — preenchido automaticamente na validação
   - Criada via `Base.metadata.create_all` (padrão do projeto — sem Alembic nesta story)

2. **AC2 — Endpoints de upload e listagem**
   - `POST /clientes/{cliente_id}/extratos` — upload de arquivo (`multipart/form-data`)
     - Salva em `extratos_clientes/{cliente_id}/{uuid}_{nome_original}`
     - Aceita extensões: `.xlsx`, `.xls`, `.csv`, `.txt`, `.zip`
     - Retorna `ExtratoClienteResponse`
   - `GET /clientes/{cliente_id}/extratos` — lista extratos do cliente com status
   - `GET /clientes/{cliente_id}/extratos/{extrato_id}/download` — retorna `FileResponse`
   - `DELETE /clientes/{cliente_id}/extratos/{extrato_id}` — remove arquivo e registro (somente `status='aguardando'`)

3. **AC3 — Validação de coerência automática**
   - `POST /clientes/{cliente_id}/extratos/validar` — dispara validação para o cliente
   - Lógica de cruzamento:
     - Para cada `ExtratoCliente` com `status='aguardando'`: busca em `Processamento` um registro com `cliente_id` igual e `nome_arquivo` contendo o `nome_arquivo` do extrato (match por substring — tolerância a prefixos de path)
     - Se encontrado e `processamento.tipo_arquivo == extrato.tipo` (ou extrato.tipo == 'Outro'): `status → 'importado'`, `processamento_id → id do processamento`
     - Se encontrado mas `tipo_arquivo` difere: `status → 'divergente'` (arquivo importado com tipo diferente do esperado)
     - Se não encontrado: mantém `'aguardando'`
   - `GET /clientes/{cliente_id}/extratos/status-resumo` — retorna contadores: `total`, `aguardando`, `importado`, `divergente`

4. **AC4 — Tela de extratos na gestão de clientes (`/clientes/[id]/extratos`)**
   - Nova rota Next.js: `apps/web/src/app/(dashboard)/clientes/[id]/extratos/page.tsx`
   - Exibe tabela de extratos do cliente: nome, tipo, data upload, status (badge colorido), ações
   - Botão "Upload Extrato" → abre modal com seleção de arquivo + campo tipo
   - Botão "Baixar" por extrato
   - Botão "Validar Coerência" → chama `/validar` e atualiza lista
   - Badges: 🔵 aguardando / 🟢 importado / 🟡 divergente
   - Link de navegação para a tela a partir da listagem de clientes existente

5. **AC5 — Qualidade**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando
   - Diretório `extratos_clientes/` adicionado ao `.gitignore`

### Model ExtratoCliente

```python
# apps/api/app/models/extrato_cliente.py
import uuid
from datetime import datetime
from sqlalchemy import Column, String, Integer, DateTime, ForeignKey
from app.models.base import Base

class ExtratoCliente(Base):
    __tablename__ = "extratos_cliente"

    id = Column(String(36), primary_key=True, default=lambda: str(uuid.uuid4()))
    cliente_id = Column(Integer, ForeignKey("clientes.cliente_id"), nullable=False, index=True)
    nome_arquivo = Column(String(255), nullable=False)
    caminho_arquivo = Column(String(500))
    tipo = Column(String(50), default="Outro")  # Venda | Recebivel | Outro
    uploaded_by = Column(String(100))
    uploaded_at = Column(DateTime, default=datetime.utcnow)
    status = Column(String(30), default="aguardando")  # aguardando | importado | divergente
    processamento_id = Column(Integer, nullable=True)
```

### Upload endpoint (multipart)

```python
from fastapi import UploadFile, File, Form

@router.post("/{cliente_id}/extratos")
async def upload_extrato(
    cliente_id: int,
    file: UploadFile = File(...),
    tipo: str = Form("Outro"),
    db: Session = Depends(get_db),
    current_user = Depends(get_current_user)
):
    ALLOWED_EXT = {".xlsx", ".xls", ".csv", ".txt", ".zip"}
    ext = Path(file.filename).suffix.lower()
    if ext not in ALLOWED_EXT:
        raise HTTPException(400, f"Extensão não permitida: {ext}")

    dest_dir = Path(f"extratos_clientes/{cliente_id}")
    dest_dir.mkdir(parents=True, exist_ok=True)
    uid = str(uuid.uuid4())
    dest_path = dest_dir / f"{uid}_{file.filename}"

    with open(dest_path, "wb") as f:
        f.write(await file.read())

    extrato = ExtratoCliente(
        cliente_id=cliente_id,
        nome_arquivo=file.filename,
        caminho_arquivo=str(dest_path),
        tipo=tipo,
        uploaded_by=current_user.login,
    )
    db.add(extrato); db.commit(); db.refresh(extrato)
    return extrato
```

### Lógica de validação (match flexível)

```python
def validar_extratos(cliente_id: int, db: Session):
    extratos = db.query(ExtratoCliente).filter(
        ExtratoCliente.cliente_id == cliente_id,
        ExtratoCliente.status == "aguardando"
    ).all()

    processamentos = db.query(Processamento).filter(
        Processamento.cliente_id == cliente_id
    ).all()

    for extrato in extratos:
        match = next((
            p for p in processamentos
            if extrato.nome_arquivo in p.nome_arquivo
            or p.nome_arquivo.endswith(extrato.nome_arquivo)
        ), None)
        if match:
            extrato.processamento_id = match.id
            # Verificar coerência de tipo
            if extrato.tipo == "Outro" or match.tipo_arquivo == extrato.tipo:
                extrato.status = "importado"
            else:
                extrato.status = "divergente"  # arquivo importado com tipo diferente

    db.commit()
```
