# Story 3.29 — Timeline de Contestações por Cliente

**Status:** Done
**Branch:** feature/nextjs-migration
**Épico:** Módulo de Contestação (continuação Story 3.12)

## Objetivo

Página `/gestao/clientes/[id]/contestacoes` com timeline visual de todas as contestações de um cliente — status, evolução, adquirente, período e valor em disputa.

## Acceptance Criteria

- [x] Endpoint `GET /contestacoes/cliente/{cliente_id}` retorna lista paginada com metadados do cliente
- [x] Timeline visual com dot colorido por status na página frontend
- [x] Status mapeados: rascunho, enviada, em_analise, deferida, indeferida
- [x] Link de navegação no `ClienteResumoPanel` → `/gestao/clientes/{id}/contestacoes`
- [x] Estado vazio com link para o módulo de contestações
- [x] ruff check: 0 erros

## Implementação

### Backend
- Novo endpoint `GET /contestacoes/cliente/{cliente_id}` em `contestacoes.py`
- Posicionado ANTES de `/{contestacao_id}` para evitar conflito de rota
- Filtra por `Contestacao.cliente_id`, ordena por `created_at DESC`
- Retorna: id, status, adquirente, processamento_id, periodo_inicio, periodo_fim, valor_excesso_total, created_at, updated_at

### Frontend
- Tipos `ContestacaoResumoItem` e `ContestacoesPorCliente` adicionados a `contestacao.ts`
- Método `porCliente(clienteId)` adicionado ao `contestacaoApi`
- Página timeline em `gestao/clientes/[id]/contestacoes/page.tsx`
- Link "Ver contestações →" adicionado ao `ClienteResumoPanel`
