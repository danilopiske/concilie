# Story 3.23 — Página de Detalhes do Processamento

## Status: Done

## Objective
Create a page `/importar/processamentos/[id]` that shows all data related to a specific processing: calculation tasks, reports and abusivity analysis — all consolidated in a single contextual view.

## Why Now
Operators need a centralized view of each processing to understand the state and result of the work performed. Currently, data is scattered across separate modules.

## Acceptance Criteria
- [x] Backend endpoint `GET /processamentos/{processamento_id}/detalhes` returns consolidated data
- [x] Endpoint includes cálculos, relatórios and abusividade tasks
- [x] Status geral is computed from task statuses (concluido / em_andamento / com_erro / parcial / sem_dados)
- [x] Frontend page `/importar/processamentos/[id]` renders all task sections
- [x] Status badges for each task item
- [x] Progress bar for tasks with progress field
- [x] Summary totals cards (importações, cálculos, relatórios, abusividade)
- [x] Back navigation to processamentos list
- [x] Loading and error states
- [x] ruff check: 0 errors

## Technical Notes
- `ImportTask` does NOT have `processamento_id` (uses `cliente_id`), so importações section always returns empty
- `CalculoTask`, `RelatorioTask`, `AbusividadeTask` all have `processamento_id`
- Endpoint added to existing `processamentos.py` router (prefix `/processamentos` already registered in api.py)
- No new router registration needed in `api.py`
