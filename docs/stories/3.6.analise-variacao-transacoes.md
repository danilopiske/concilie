## Story

**As a** usuário analisando um processamento no Concilie,
**I want** visualizar na tela de análise de cálculos quais períodos (meses) tiveram ausência total ou volume muito reduzido de transações,
**so that** eu consiga identificar rapidamente lacunas de dados (arquivos não importados, meses sem movimento ou inconsistência na base) antes de interpretar os resultados do cálculo de taxas.

## Acceptance Criteria

1. **AC1 — Endpoint `GET /calculos/analise-periodos/{processamento_id}`**
   - Retorna lista de períodos (meses) dentro do intervalo `min(data_venda)` → `max(data_venda)` do processamento
   - Para cada mês: `periodo` (YYYY-MM), `quantidade` (int), `valor_total` (float), `status` (`ok` | `reduzido` | `ausente`)
   - Lógica de classificação:
     - `ausente`: mês no intervalo com `quantidade == 0`
     - `reduzido`: `quantidade < 50%` da mediana dos meses com movimento
     - `ok`: demais
   - Usa tabela `vendas_calculos` se existir para o processamento (cálculo realizado), senão `vendas_processadas`
   - Parâmetro opcional `?threshold=0.5` para ajustar o percentual de corte (default 50%)

2. **AC2 — Schema de resposta Pydantic**
   - `PeriodoAnalise`: `periodo: str`, `quantidade: int`, `valor_total: float`, `status: Literal["ok", "reduzido", "ausente"]`
   - `AnalisePeriodosResponse`: `processamento_id: str`, `total_periodos: int`, `periodos_ausentes: int`, `periodos_reduzidos: int`, `mediana_quantidade: float`, `periodos: List[PeriodoAnalise]`

3. **AC3 — Painel "Análise de Períodos" na tela de cálculos (`calculos/page.tsx`)**
   - Exibido após seleção de processamento (mesmo sem ter calculado)
   - Botão "Analisar Períodos" separado dos botões Preview/Calcular
   - Mostra mini-cards de resumo: total de períodos, ausentes, reduzidos
   - Tabela/lista dos períodos com badge colorido: 🟢 ok / 🟡 reduzido / 🔴 ausente
   - Loading state enquanto busca; sem dados se processamento não selecionado

4. **AC4 — Alerta visual quando houver anomalias**
   - Se `periodos_ausentes > 0` ou `periodos_reduzidos > 0`: mostrar banner de aviso amarelo acima dos resultados do cálculo
   - Exemplo: "⚠️ 2 períodos ausentes e 1 período com volume reduzido detectados. Verifique os dados antes de interpretar o resultado."
   - Banner só aparece após clicar "Analisar Períodos"

5. **AC5 — Qualidade**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando
   - Endpoint documentado no Swagger (`/docs`)

### Agrupamento por mês (SQLite-compatível)

```python
from sqlalchemy import func

rows = (
    db.query(
        func.strftime('%Y-%m', model.data_venda).label("periodo"),
        func.count().label("quantidade"),
        func.sum(model.valor_venda).label("valor_total")
    )
    .filter(model.processamento_id == processamento_id)
    .group_by(func.strftime('%Y-%m', model.data_venda))
    .order_by("periodo")
    .all()
)
```

### Geração do intervalo completo (sem gap)

```python
from datetime import date
import statistics

# Gerar todos os meses entre min e max
def gerar_meses(min_periodo: str, max_periodo: str) -> list[str]:
    from dateutil.relativedelta import relativedelta
    start = date.fromisoformat(min_periodo + "-01")
    end = date.fromisoformat(max_periodo + "-01")
    meses = []
    cur = start
    while cur <= end:
        meses.append(cur.strftime('%Y-%m'))
        cur += relativedelta(months=1)
    return meses

# Classificar
dados_map = {r.periodo: r for r in rows}
quantidades = [r.quantidade for r in rows if r.quantidade > 0]
mediana = statistics.median(quantidades) if quantidades else 0

resultado = []
for mes in gerar_meses(min_periodo, max_periodo):
    if mes not in dados_map:
        status = "ausente"
        qtd, total = 0, 0.0
    elif dados_map[mes].quantidade < mediana * threshold:
        status = "reduzido"
        qtd, total = dados_map[mes].quantidade, float(dados_map[mes].valor_total or 0)
    else:
        status = "ok"
        qtd, total = dados_map[mes].quantidade, float(dados_map[mes].valor_total or 0)
    resultado.append(PeriodoAnalise(periodo=mes, quantidade=qtd, valor_total=total, status=status))
```
