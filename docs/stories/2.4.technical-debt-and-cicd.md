## Story

**As a** desenvolvedor/mantenedor do sistema Concilie,
**I want** ter a dívida técnica mapeada, o pipeline CI/CD hardened e os padrões de qualidade de código estabelecidos,
**so that** o projeto esteja em estado saudável para desenvolvimento contínuo e deploys seguros.

## Acceptance Criteria

1. **AC1 — Pipeline CI/CD unificado**
   - Existe `.github/workflows/ci.yml` que roda em PRs e pushes para `main`
   - Pipeline executa: lint → typecheck → pytest → build
   - Falhas em qualquer step bloqueiam o merge
   - Cache de dependências configurado (npm + pip) para reduzir tempo de CI

2. **AC2 — Variáveis de ambiente e segredos**
   - Todas as variáveis sensíveis estão em `.env.example` (sem valores reais)
   - GitHub Secrets configurados para CI: `DATABASE_URL`, `SECRET_KEY`, `JWT_SECRET`
   - Nenhum arquivo `.env` com dados reais commitado no repositório
   - `apps/api/.env.example` e `apps/web/.env.example` atualizados e documentados

3. **AC3 — Linting e formatação padronizados**
   - `ruff` configurado em `apps/api/pyproject.toml` ou `ruff.toml`
   - `eslint` + `prettier` configurados em `apps/web/`
   - Pre-commit hooks configurados (`.pre-commit-config.yaml`) ou documentado que hooks são opcionais
   - `npm run lint` e `ruff check app/` passam sem erros no estado atual

4. **AC4 — Dívida técnica mapeada**
   - Arquivo `docs/technical-debt.md` criado com inventário das dívidas identificadas
   - Cada item tem: descrição, impacto (H/M/L), esforço estimado, owner sugerido
   - Itens críticos (impacto H) têm issue ou story de follow-up referenciada
   - Dívidas de Story 2.1 (backend), 2.2 (frontend) e 2.3 (testes) consolidadas aqui

5. **AC5 — Testes de arquivos temporários**
   - Arquivos de teste avulsos na raiz (`test_api_connection.py`, `test_db.py`, etc.) são movidos ou removidos
   - `apps/api/` sem arquivos `test_*.py` fora de `apps/api/tests/`
   - Raiz do monorepo sem arquivos de teste soltos

6. **AC6 — Documentação mínima de operação**
   - `README.md` na raiz do monorepo atualizado com: pré-requisitos, como rodar localmente, como rodar testes
   - `apps/api/README.md` com instruções de setup do backend
   - `apps/web/README.md` com instruções de setup do frontend
   - Variáveis de ambiente documentadas nos respectivos `.env.example`

### Estrutura CI/CD esperada

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: 'pip'
      - run: pip install -r apps/api/requirements.txt
      - run: cd apps/api && ruff check app/
      - run: cd apps/api && pytest tests/ -v

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: apps/web/package-lock.json
      - run: cd apps/web && npm ci
      - run: cd apps/web && npm run lint
      - run: cd apps/web && npm run typecheck
      - run: cd apps/web && npm run build
```
