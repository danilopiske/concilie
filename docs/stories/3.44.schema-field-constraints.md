# Story 3.44 — Schema Field Constraints (Pydantic Validation)

**Status:** Done
**Branch:** feature/nextjs-migration

## Story
Como desenvolvedor, quero que os schemas Pydantic da API tenham constraints de `max_length` nos campos string, para que entradas inválidas sejam rejeitadas na camada de validação antes de atingir o banco de dados.

## Acceptance Criteria
- [x] `ClienteBase` e `ClienteCreate`: `nome_fantasia` max 200, `razao_social` max 200, `cnpj` max 18
- [x] `ClienteUpdate`: mesmos constraints opcionais
- [x] `ExtratoClienteCreate.tipo`: default "Outro", max 50
- [x] `CalculoPreviewRequest.processamento_id`: max 100
- [x] `CalculoPreviewRequest.tipo_taxa`: max 50
- [x] `CalculoHistoryItem.calc_id`: max 100, `calc_tipo`: max 50, `calc_usuario`: max 100
- [x] `ContestacaoSaveEdit.html_content`: max 500_000
