# Story 3.26 — Relatório de Divergências por Cliente

## Status: Done

## Objetivo
Endpoint e página que consolidam divergências encontradas para um cliente específico — comparando taxas contratadas vs cobradas em todos os ECs vinculados — e permitem exportar como CSV.

## Acceptance Criteria
- [x] GET `/api/v1/divergencias/{cliente_id}` retorna lista de divergências por bandeira/modalidade
- [x] GET `/api/v1/divergencias/{cliente_id}/exportar-csv` retorna CSV para download
- [x] Lógica usa relação `ECCliente` para mapear `cliente_id → ec_id → Taxa`
- [x] Comparação por `bandeira` + `modalidade/forma_pagamento`
- [x] Nota informativa quando cliente não possui ECs vinculados
- [x] Página `/gestao/clientes/[id]/divergencias` com KPIs e tabela
- [x] Botão "Exportar CSV" visível apenas quando há divergências
- [x] Link "Ver relatório de divergências →" no `ClienteResumoPanel`
- [x] ruff check: 0 erros

## Implementação

### Schema real utilizado
- `TaxaContratada`: `cliente_id`, `bandeira`, `modalidade`, `taxa_contratada`
- `Taxa`: `ec` (código EC), `bandeira`, `forma_pagamento`, `taxa`
- Relação: `ECCliente` (`cliente_id` ↔ `ec_id`) → join com `Taxa.ec`

### Lógica de comparação
- Por cada `TaxaContratada` do cliente, busca `Taxa` com mesmo `bandeira` + `forma_pagamento`
- Usa a maior taxa cobrada encontrada como referência
- Registra divergência quando `max(taxa_cobrada) > taxa_contratada`
