# Story 3.28: Ranking de Recuperação Financeira por Cliente

**Status:** Done
**Epic:** 3 — Análise Financeira
**Branch:** feature/nextjs-migration

## Objetivo

Página `/gestao/recuperacao` mostrando ranking de clientes ordenados pelo valor total de perdas financeiras contestáveis (`VendasCalculos.perda`), permitindo priorizar quais clientes atacar primeiro via contestação.

## Acceptance Criteria

- [x] Endpoint `GET /api/v1/recuperacao/ranking` retorna clientes rankeados por `sum(perda)` desc
- [x] Endpoint `GET /api/v1/recuperacao/ranking/exportar-csv` exporta ranking em CSV
- [x] Página `/gestao/recuperacao` exibe KPIs: total recuperável e número de clientes com perdas
- [x] Ranking lista clientes com posição, nome, total perda, barra proporcional, link para detalhes
- [x] Sidebar "Gestão" inclui item "Recuperação Financeira" com ícone TrendingUp
- [x] Ruff check: 0 erros

## Decisões Técnicas

### Schema Real Encontrado

- `VendasCalculos` (tabela: `vendas_calculos`) — possui `perda` (DECIMAL 18,2), `ec_id` (BigInteger), `id_venda`, `calc_id`
- **Não há `cliente_id` direto em `VendasCalculos`**
- Join strategy: `VendasCalculos.ec_id` → `cast(String)` → `ECCliente.ec_id` → `ECCliente.cliente_id` → `Cliente`
- Fallback: agrupar por `ec_id` diretamente se join retornar vazio

### Sem `CalculoTask.cliente_id`

`CalculoTask` tem apenas `processamento_id` — não foi usada no join. A rota mais direta foi via `ECCliente`.
