## Story

**As a** operador do Concilie,
**I want** receber notificações in-app quando eventos importantes ocorrem (processamento concluído, abusividade detectada, extrato divergente) e visualizar um centro de alertas com histórico,
**so that** eu seja proativamente informado sobre situações que requerem ação, sem precisar ficar atualizando as telas manualmente.

## Acceptance Criteria

1. **AC1 — Model `Notificacao`**
   - Nova tabela `notificacoes`:
     - `id` (UUID, PK)
     - `usuario_id` (Integer, FK → usuarios, nullable) — `Optional[int]` para suportar notificações de sistema sem usuário autenticado (contrato @architect)
     - `tipo` (String 50): `importacao_ok` | `importacao_erro` | `calculo_ok` | `relatorio_ok` | `abusividade_detectada` | `extrato_divergente`
     - `titulo` (String 200)
     - `mensagem` (String 500)
     - `link` (String 500, nullable) — rota frontend para navegar ao clicar
     - `lida` (Boolean, default=False)
     - `created_at` (DateTime, default=now)
   - Criada via `Base.metadata.create_all`

2. **AC2 — NotificacaoService e pontos de disparo**
   - `NotificacaoService.criar(usuario_id: Optional[int] = None, tipo, titulo, mensagem, link)` — insere na tabela; `usuario_id=None` permitido para eventos de sistema
   - Pontos de disparo (adicionar chamada ao `NotificacaoService` nos services existentes):
     - `import_service.py` — ao finalizar importação: `importacao_ok` ou `importacao_erro`
     - `calculo_service.py` — ao concluir cálculo: `calculo_ok`
     - `relatorio_service.py` — ao concluir relatório: `relatorio_ok`
     - `abusividade_service.py` — quando `status == "critico"` detectado: `abusividade_detectada`
     - `extratos_cliente.py` (Story 3.7) — ao validar extrato com status `divergente`: `extrato_divergente`
   - `usuario_id` obtido do contexto de execução — passar como parâmetro nos services

3. **AC3 — Endpoints de notificações**
   - `GET /notificacoes` — lista notificações do usuário logado, ordenadas por `created_at` desc, com paginação `?limit=20&offset=0`
   - `GET /notificacoes/nao-lidas/count` — retorna `{"count": N}` — endpoint leve para polling
   - `PUT /notificacoes/{id}/lida` — marca como lida
   - `PUT /notificacoes/marcar-todas-lidas` — marca todas do usuário como lidas
   - `DELETE /notificacoes/{id}` — remove
   - Todas as rotas requerem autenticação JWT

4. **AC4 — Sino de notificações no header**
   - Ícone `Bell` (lucide-react) no header com badge vermelho mostrando count não lidas
   - Polling `GET /notificacoes/nao-lidas/count` a cada 30s via `setInterval`
   - Clique no sino: abre dropdown com últimas 5 notificações + link "Ver todas"
   - Cada notificação no dropdown: ícone de tipo + título + tempo relativo ("há 5 min") + indicador de não lida
   - Clique na notificação: navega para `link` da notificação + marca como lida

5. **AC5 — Centro de alertas `/notificacoes`**
   - Página completa com lista de todas as notificações do usuário
   - Filtros: por tipo, por status (lida/não lida)
   - Botão "Marcar todas como lidas"
   - Paginação
   - Link "Notificações" no sidebar (seção Configurações ou footer)

6. **AC6 — Qualidade**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando
   - Teste unitário: `tests/unit/test_notificacao_service.py` — criar notificação + marcar lida

## Dev Notes

### Passar usuario_id para os services

O `current_user` está disponível nos endpoints via `Depends(get_current_user)`. Para services assíncronos (background tasks), o `usuario_id` deve ser capturado no endpoint e passado como parâmetro:

```python
# No endpoint:
background_tasks.add_task(import_service.processar, task_id, usuario_id=current_user.id)

# No service:
async def processar(task_id: str, usuario_id: int, db: Session):
    # ... lógica ...
    NotificacaoService.criar(db, usuario_id, "importacao_ok", "Importação concluída", ...)
```

### Tempo relativo no frontend

```typescript
function tempoRelativo(date: string): string {
  const diff = Date.now() - new Date(date).getTime();
  const minutos = Math.floor(diff / 60000);
  if (minutos < 1) return "agora";
  if (minutos < 60) return `há ${minutos} min`;
  const horas = Math.floor(minutos / 60);
  if (horas < 24) return `há ${horas}h`;
  return `há ${Math.floor(horas / 24)}d`;
}
```

useEffect(() => {
  const poll = () => notificacaoApi.getCount().then(setCount);
