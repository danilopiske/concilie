## Story

**As a** administrador do sistema Concilie,
**I want** todos os endpoints da API protegidos por autenticação JWT,
**so that** apenas usuários autenticados possam acessar os dados financeiros e funcionalidades do sistema.

## Acceptance Criteria

1. **AC1 — Proteção global via `api.py`**
   - Todos os routers registrados em `app/api/v1/api.py` (exceto `login`) recebem `dependencies=[Depends(deps.get_current_user)]` no `include_router`
   - Requisições sem token válido retornam `401 Unauthorized`
   - Endpoint público mantido: `POST /api/v1/login/access-token` e `POST /api/v1/logout`
   - Endpoint público mantido: `GET /api/v1/health` (se existir)

2. **AC2 — Fixture `auth_headers` funcional nos testes**
   - `conftest.py` corrigido: senha do usuário de teste criada com `security.get_password_hash("admin123")` (não plaintext)
   - Fixture `auth_headers` retorna headers válidos com JWT
   - Todos os testes que chamam endpoints protegidos recebem `auth_headers`

3. **AC3 — Testes passando após proteção**
   - `pytest tests/ -v` → todos os testes que passavam antes continuam passando
   - Testes que testam endpoints protegidos usam `auth_headers`
   - Nenhum teste novo falha por `401`

4. **AC4 — Proteção por-endpoint removida onde redundante**
   - Endpoints em `relatorios.py`, `usuarios.py`, `relatorio_tags.py` que já tinham `Depends(get_current_user)` inline podem manter ou remover o inline (não duplicar desnecessariamente — decisão do dev)
   - Consistência: não misturar estilos no mesmo arquivo sem motivo

5. **AC5 — Lint e ruff continuam passando**
   - `ruff check app/` → 0 erros após as mudanças

### Contexto

A `deps.py` já tem `get_current_user` completamente implementado:
- Aceita token via **Bearer header** (`Authorization: Bearer <token>`) OU via **cookie HttpOnly** (`access_token`)
- Valida JWT com `settings.SECRET_KEY` e `settings.ALGORITHM`
- Retorna objeto `Usuario` do banco
- Lança `401 Unauthorized` se token ausente, `403 Forbidden` se token inválido

### Abordagem recomendada — proteção em `api.py`

Ao invés de editar 13+ arquivos de endpoint, centralizar em `api.py`:

```python
# api.py — ANTES
api_router.include_router(clientes.router, prefix="/clientes", tags=["clientes"])

# api.py — DEPOIS
from fastapi import Depends
from app.api import deps

api_router.include_router(
    clientes.router,
    prefix="/clientes",
    tags=["clientes"],
    dependencies=[Depends(deps.get_current_user)],
)
```

Aplicar este padrão para TODOS os routers exceto `login.router`.

### Problema na fixture `auth_headers`

O conftest.py atual cria o usuário de teste com senha em plaintext:
```python
# ATUAL (quebrado) — senha não é hash bcrypt, login falha
db.add(Usuario(usuario="admin@test.com", senha="admin123", nome="Admin Test"))

# CORRETO — usar hash bcrypt igual ao que o sistema usa em produção
from app.core import security
db.add(Usuario(usuario="admin@test.com", senha=security.get_password_hash("admin123"), nome="Admin Test"))
```

### Testes que precisam de `auth_headers`

Auditar cada arquivo de teste. Padrão de atualização:

```python
# ANTES
def test_listar_clientes(client):
    response = client.get("/api/v1/clientes/")
    assert response.status_code == 200

# DEPOIS
def test_listar_clientes(client, auth_headers):
    response = client.get("/api/v1/clientes/", headers=auth_headers)
    assert response.status_code == 200
```

### Testes que devem continuar SEM auth (públicos)

- `test_health.py` — `/api/v1/health` deve ser público
- `test_auth.py` — login/logout são públicos por natureza
- Qualquer teste de `401` (validar que endpoint SEM token retorna 401)
