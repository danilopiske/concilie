## Dev Notes

### Situação atual (pré-Story 3.2)

**`config.py` — CORS:**
```python
# Atual: ALLOWED_ORIGINS é lista hardcoded, CORS_ORIGINS é property dinâmica
ALLOWED_ORIGINS: List[str] = [
    "http://localhost:3000",
    "http://localhost:8000",
    "http://127.0.0.1:3000",
    "http://127.0.0.1:8000",
]

@property
def CORS_ORIGINS(self) -> List[str]:
    origins = self.ALLOWED_ORIGINS.copy()
    if os.getenv("FRONTEND_URL"):
        origins.append(os.getenv("FRONTEND_URL"))
    if os.getenv("RAILWAY_PUBLIC_DOMAIN"):
        origins.append(f"https://{os.getenv('RAILWAY_PUBLIC_DOMAIN')}")
    return origins
```

O problema (B-04): não há como sobrescrever toda a lista por variável de ambiente. Em produção, para restringir a apenas `["https://app.concilie.com.br"]`, precisaria remover os defaults do código.

**Abordagem recomendada para AC1:**
```python
# ADICIONAR ao Settings:
ALLOWED_ORIGINS_STR: Optional[str] = None  # JSON string, ex: '["https://app.com"]'

@property
def CORS_ORIGINS(self) -> List[str]:
    if self.ALLOWED_ORIGINS_STR:
        import json
        return json.loads(self.ALLOWED_ORIGINS_STR)
    # fallback: comportamento atual
    origins = self.ALLOWED_ORIGINS.copy()
    if os.getenv("FRONTEND_URL"):
        origins.append(os.getenv("FRONTEND_URL"))
    return origins
```

### Estrutura Dockerfile recomendada

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# Instalar Poetry
RUN pip install poetry==1.8.3

# Copiar dependências primeiro (cache layer)
COPY pyproject.toml poetry.lock ./
RUN poetry config virtualenvs.create false \
    && poetry install --only main --no-interaction

# Copiar código
COPY . .

EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Nota:** `--only main` em vez de `--no-dev` (Poetry v1.8.x usa `--only main`)

#### Security Assessment

- **Secrets no código:** `SECRET_KEY` tem fallback explicitamente nomeado "dev-secret-key-change-in-production-only-for-local" — aceitável para dev ✅
- **CI secrets:** fallback `||` no GitHub Actions é syntax válida ✅
- **Docker:** `.dockerignore` exclui `.env`, `.env.*` corretamente ✅
- **CORS:** override via env var impede abertura acidental em produção ✅

#### Concerns (não bloqueadores)

- **CONCERN-1 (MEDIUM):** `config.py` linha 41: `json.loads(self.ALLOWED_ORIGINS_STR)` sem `try/except JSONDecodeError`. JSON inválido em produção causará crash na inicialização da aplicação. Sugestão: envolver em try/except com fallback para lista default e log de warning.
- **CONCERN-2 (LOW):** `ci.yml` linha 46 mantém `CORS_ORIGINS: '["http://localhost:3000"]'` no env do pytest. Este env var é ignorado — `config.py` lê `ALLOWED_ORIGINS_STR`, não `CORS_ORIGINS`. Pode causar confusão. Sugestão: remover ou substituir por `ALLOWED_ORIGINS_STR`.
- **CONCERN-3 (LOW):** Task 4.4 (docker build) não foi executada — Docker Desktop não disponível durante implementação. Recomendado executar antes do primeiro deploy.

#### Recommendation

Story 3.2 resolve corretamente as dívidas técnicas B-04 e C-01. A implementação de CORS por env var é funcional. O concern MEDIUM (json.loads sem tratamento de erro) é o único item que merece atenção antes de ir para produção, mas não bloqueia o merge desta branch pois o valor será controlado pela equipe.
