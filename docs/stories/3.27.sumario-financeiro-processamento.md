# Story 3.27 — Sumário Financeiro por Processamento (Valor em R$)

**Status:** Done
**Sprint:** 3
**Epic:** Gestão de Processamentos

## Objetivo

Adicionar um endpoint de sumário financeiro que mostra o valor monetário real (R$) das divergências detectadas por processamento, permitindo que operadores priorizem contestações pelo impacto financeiro.

## Acceptance Criteria

- [x] Endpoint `GET /processamentos/{processamento_id}/financeiro` retorna sumário financeiro
- [x] Retorna total de vendas (R$), taxas cobradas (R$), taxas contratadas (R$), diferença (R$)
- [x] Retorna contagem de transações e taxas médias (%)
- [x] Interface TypeScript `SumarioFinanceiro` adicionada ao API client
- [x] Página de detalhes do processamento exibe card de sumário financeiro
- [x] Card só aparece quando há dados reais (`tem_dados: true`)
- [x] ruff check: 0 erros

## Implementação

### Modelo de Dados Utilizado

`VendasCalculos` (`vendas_calculos`) — modelo central com:
- `calc_id`: liga ao `processamento_id`
- `vl_venda`: valor bruto da venda
- `desc_venda`: desconto cobrado pelo adquirente (taxa cobrada R$)
- `desc_calc`: desconto calculado pelo sistema (taxa contratada R$)
- `perda`: diferença financeira (valor contestável)

### Files Modified

- `apps/api/app/api/v1/endpoints/processamentos.py` — novo endpoint `/{processamento_id}/financeiro`
- `apps/web/src/lib/api/processamentos.ts` — interface `SumarioFinanceiro` + `getSumarioFinanceiro()`
- `apps/web/src/app/(dashboard)/importar/processamentos/[id]/page.tsx` — card de sumário financeiro

```json
}
```
