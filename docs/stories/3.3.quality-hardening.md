## Acceptance Criteria

1. **AC1 — `config.py`: json.loads protegido contra JSON inválido**
   - `CORS_ORIGINS` property envolve `json.loads(self.ALLOWED_ORIGINS_STR)` em `try/except json.JSONDecodeError`
   - Em caso de JSON inválido: logar warning e retornar `self.ALLOWED_ORIGINS.copy()` como fallback seguro
   - Aplicação não crasha na inicialização com valor inválido em `ALLOWED_ORIGINS_STR`
   - QA CONCERN-1 (Story 3.2) marcado como resolvido

2. **AC2 — `security.py`: ALGORITHM centralizado e datetime atualizado**
   - `ALGORITHM = "HS256"` hardcoded removido de `security.py` — usar exclusivamente `settings.ALGORITHM`
   - `datetime.utcnow()` substituído por `datetime.now(UTC)` (compatível com Python 3.12+)
   - QA CONCERN-1 e CONCERN-3 (Story 3.1) resolvidos

3. **AC3 — `conftest.py`: `auth_headers` falha explicitamente**
   - `auth_headers` fixture lança `pytest.fail(...)` quando login retorna status != 200, em vez de retornar `{}`
   - Testes que dependem de `auth_headers` falham de forma clara e rastreável
   - Todos os 52 testes continuam passando
   - QA CONCERN-2 (Story 3.1) resolvido

4. **AC4 — `relatorio_service.py`: bare `except` e acoplamento corrigidos**
   - `except:` (bare) na linha 186 substituído por `except Exception as e:` com logging adequado
   - Acoplamento direto `Session(engine)` em `run_async_report` e `update_task_progress` documentado com TODO para refatoração futura (B-01 parcial)
   - Ruff 0 erros após alterações

5. **AC5 — `depara.py`: sys.path hardcoded removido**
   - Endpoint `/ler-cabecalhos` em `depara.py` remove fallback `sys.path.append('d:/Financial  base/...')` hardcoded
   - Lógica de fallback usa caminho relativo calculado dinamicamente ou lança erro claro se módulo não encontrado
   - Dívida B-06 marcada como `resolved` em `docs/technical-debt.md`

6. **AC6 — `ci.yml`: env var CORS_ORIGINS legada removida**
   - Remover linha `CORS_ORIGINS: '["http://localhost:3000"]'` do step `Run pytest` em `ci.yml` (env var ignorada — config.py usa `ALLOWED_ORIGINS_STR`)
   - QA CONCERN-2 (Story 3.2) resolvido

7. **AC7 — Todos os testes passando**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → 52/52 passando (ou mais, se novos testes adicionados)

### Task 1 — Implementação correta (config.py)

```python
# ANTES (crash em produção com JSON inválido):
if self.ALLOWED_ORIGINS_STR:
    return json.loads(self.ALLOWED_ORIGINS_STR)

# DEPOIS (safe com fallback):
import logging
_logger = logging.getLogger(__name__)

if self.ALLOWED_ORIGINS_STR:
    try:
        return json.loads(self.ALLOWED_ORIGINS_STR)
    except json.JSONDecodeError as e:
        _logger.warning(
            "ALLOWED_ORIGINS_STR inválido (%s), usando defaults. Valor: %r",
            e, self.ALLOWED_ORIGINS_STR
        )
```

### Task 2 — Implementação correta (security.py)

```python
# ANTES:
from datetime import datetime, timedelta
ALGORITHM = "HS256"
...
expire = datetime.utcnow() + timedelta(...)
encoded_jwt = jwt.encode(to_encode, settings.SECRET_KEY, algorithm=ALGORITHM)

# DEPOIS:
from datetime import datetime, timedelta, timezone
# ALGORITHM removido — usar settings.ALGORITHM
...
expire = datetime.now(timezone.utc) + timedelta(...)
encoded_jwt = jwt.encode(to_encode, settings.SECRET_KEY, algorithm=settings.ALGORITHM)
```
