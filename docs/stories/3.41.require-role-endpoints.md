# Story 3.41 — Aplicar require_role() em Todos os Endpoints

## Status: Done

## Contexto

A Story 3.39 criou a infraestrutura de controle de acesso (`require_role()`, modelos, perfis). Porém, apenas 2 endpoints usam `require_role()`. Os outros 32 endpoints autenticados não verificam perfil — qualquer usuário logado (incluindo `visualizador`) pode executar operações críticas.

Esta story completa a fase 2 da 3.39: aplicar `require_role()` em todos os endpoints que necessitam de restrição de perfil.

---

## Story

**Como** administrador do sistema,
**Quero** que cada endpoint verifique o perfil do usuário,
**Para** garantir que visualizadores não modifiquem dados e operadores não acessem configurações.

---

## Acceptance Criteria

### AC1 — Endpoints admin-only protegidos
- [x] `auditoria.py` — todos os endpoints: `require_role(["admin"])`
- [x] `sistema.py` — endpoints de configuração: `require_role(["admin"])`
- [x] `usuarios.py` — todos os endpoints (GET, POST, PUT, DELETE): `require_role(["admin"])`
- [x] `configuracoes` — qualquer endpoint de configuração global: `require_role(["admin"])`

### AC2 — Endpoints operador+admin protegidos
- [x] `importacao_async.py` — upload e processamento: `require_role(["admin", "operador"])`
- [x] `gestao.py` — operações de escrita (POST, PUT, DELETE): `require_role(["admin", "operador"])`
- [x] `calculos` — criação e execução de cálculos: `require_role(["admin", "operador"])`
- [x] `contestacoes.py` — criação e atualização: `require_role(["admin", "operador"])`
- [x] `correcao.py` — operações de correção: `require_role(["admin", "operador"])`
- [x] `depara.py` — operações de escrita: `require_role(["admin", "operador"])`
