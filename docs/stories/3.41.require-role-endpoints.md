# Story 3.41 — Aplicar require_role() em Todos os Endpoints

## Status: Ready

## Contexto

A Story 3.39 criou a infraestrutura de controle de acesso (`require_role()`, modelos, perfis). Porém, apenas 2 endpoints usam `require_role()`. Os outros 32 endpoints autenticados não verificam perfil — qualquer usuário logado (incluindo `visualizador`) pode executar operações críticas.

Esta story completa a fase 2 da 3.39: aplicar `require_role()` em todos os endpoints que necessitam de restrição de perfil.

---

## Story

**Como** administrador do sistema,
**Quero** que cada endpoint verifique o perfil do usuário,
**Para** garantir que visualizadores não modifiquem dados e operadores não acessem configurações.

---

## Acceptance Criteria

### AC1 — Endpoints admin-only protegidos
- [ ] `auditoria.py` — todos os endpoints: `require_role(["admin"])`
- [ ] `sistema.py` — endpoints de configuração: `require_role(["admin"])`
- [ ] `usuarios.py` — todos os endpoints (GET, POST, PUT, DELETE): `require_role(["admin"])`
- [ ] `configuracoes` — qualquer endpoint de configuração global: `require_role(["admin"])`

### AC2 — Endpoints operador+admin protegidos
- [ ] `importacao_async.py` — upload e processamento: `require_role(["admin", "operador"])`
- [ ] `gestao.py` — operações de escrita (POST, PUT, DELETE): `require_role(["admin", "operador"])`
- [ ] `calculos` — criação e execução de cálculos: `require_role(["admin", "operador"])`
- [ ] `contestacoes.py` — criação e atualização: `require_role(["admin", "operador"])`
- [ ] `correcao.py` — operações de correção: `require_role(["admin", "operador"])`
- [ ] `depara.py` — operações de escrita: `require_role(["admin", "operador"])`

### AC3 — Endpoints de leitura (todos os perfis)
- [ ] `dashboard.py` — GET endpoints: `require_role(["admin", "operador", "visualizador"])`
- [ ] `relatorios.py` — GET endpoints: `require_role(["admin", "operador", "visualizador"])`
- [ ] `analise.py` — GET endpoints: `require_role(["admin", "operador", "visualizador"])`
- [ ] `abusividade.py` — GET endpoints: `require_role(["admin", "operador", "visualizador"])`

### AC4 — Endpoints públicos documentados
- [ ] `login.py` — sem autenticação (INTENCIONAL — documentar no código)
- [ ] `recuperacao.py` — confirmar se deve ser público ou protegido; documentar decisão
- [ ] `termos.py` — confirmar se deve ser público; documentar decisão

### AC5 — Regressão
- [ ] Todos os 62 testes unitários existentes continuam passando
- [ ] Nenhum endpoint anteriormente funcional quebra

### AC6 — Testes
- [ ] Teste: visualizador recebe 403 em endpoint admin-only
- [ ] Teste: operador recebe 403 em endpoint admin-only
- [ ] Teste: operador acessa endpoint operador+admin com sucesso
