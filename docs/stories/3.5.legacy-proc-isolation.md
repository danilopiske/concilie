## Story

**As a** desenvolvedor mantendo o sistema Concilie,
**I want** o acoplamento direto com `proc.proc_importacao` encapsulado atrás de um adaptador interno em `app/`,
**so that** `sys.path.insert` seja removido de `main.py`, a testabilidade do `ImportService` melhore, e a base esteja preparada para a migração completa L-01 sem quebrar o sistema.

## Contexto Técnico

`reconciliation_core.py` já foi migrado em Sprint 2 (sem dependência legada — B-02 parcialmente resolvido).
O acoplamento restante está em:

| Arquivo | Dependência legada | Tipo |
|---------|-------------------|------|
| `app/services/import_service.py` | `from proc.proc_importacao import classificar_e_gravar_recebiveis` | import direto (linha 15) |
| `app/services/import_service.py` | `from proc.proc_importacao import normalizar_dataframe_recebiveis` | lazy import (linha 175) |
| `app/services/import_service.py` | `from proc.proc_importacao import normalizar_dataframe_vendas` | lazy import (linha 180) |
| `app/api/v1/endpoints/depara.py` | `from proc.proc_importacao import is_multisheet_rede_file, ...` | lazy import com sys.path (linha 87-89) |
| `app/main.py` | `sys.path.insert(0, str(root_dir))` | necessário para tudo acima |

## Acceptance Criteria

1. **AC1 — Adaptador `app/adapters/proc_importacao_adapter.py`**
   - Criar `apps/api/app/adapters/` (novo diretório)
   - Criar `proc_importacao_adapter.py` com funções wrapper que delegam para `proc.proc_importacao`:
     - `classificar_e_gravar_recebiveis(*args, **kwargs)`
     - `normalizar_dataframe_recebiveis(*args, **kwargs)`
     - `normalizar_dataframe_vendas(*args, **kwargs)`
     - `is_multisheet_rede_file(*args, **kwargs)` (usada em `depara.py`)
   - O adaptador importa `proc.proc_importacao` em um único ponto — eliminando importações espalhadas
   - Docstring no topo explica: "Adapter temporário — será removido quando L-01 migrar proc/ para app/"

2. **AC2 — `import_service.py` usa o adaptador**
   - Substituir `from proc.proc_importacao import ...` pelo adaptador:
     `from app.adapters.proc_importacao_adapter import classificar_e_gravar_recebiveis, normalizar_dataframe_recebiveis, normalizar_dataframe_vendas`
   - Remover os lazy imports inline de `proc.proc_importacao` dentro de funções
   - Comportamento funcional idêntico ao atual

3. **AC3 — `endpoints/depara.py` usa o adaptador**
   - Substituir bloco lazy `sys.path.append` + `from proc.proc_importacao import is_multishift_rede_file, ...` (linhas 86-90)
   - Usar import direto do adaptador no topo do arquivo
   - `sys.path.append` local removido do endpoint

4. **AC4 — `main.py` sys.path simplificado**
   - Verificar se `sys.path.insert` em `main.py` ainda é necessário após AC1-AC3
   - Se `relatorios.py` ainda usa `modules.reports` via sys.path: manter sys.path mas remover o comentário referente a `import_service` e `reconciliation_core`
   - Atualizar comentário TODO para refletir o que ainda depende de sys.path

5. **AC5 — B-02 marcado como resolved**
   - `reconciliation_core.py` já migrado (Sprint 2) + `import_service.py` isolado via adapter
   - Marcar `B-02` como `resolved` em `docs/technical-debt.md`

6. **AC6 — Qualidade**
   - `ruff check app/` → 0 erros
   - `pytest tests/ -v` → todos passando (incluindo os novos se houver)

### Implementação do adaptador

```python
"""
Adapter temporário para proc.proc_importacao.
Centraliza o acoplamento legado em um único ponto.
Será removido quando L-01 migrar proc/ → app/ completamente.
"""
from proc.proc_importacao import (
    classificar_e_gravar_recebiveis,
    normalizar_dataframe_recebiveis,
    normalizar_dataframe_vendas,
    is_multisheet_rede_file,
)

__all__ = [
    "classificar_e_gravar_recebiveis",
    "normalizar_dataframe_recebiveis",
    "normalizar_dataframe_vendas",
    "is_multisheet_rede_file",
]
```
