# Story 3.37 — Relatórios: Histórico com Filtros e Auto-refresh

**Status:** Done
**Branch:** feature/nextjs-migration
**Epic:** 3 — Módulo de Relatórios

---

## Objetivo

Melhorar a página de gestão de relatórios com filtros por status/tipo e auto-refresh automático para tarefas em andamento.

## Acceptance Criteria

- [x] `GET /relatorios/historico` aceita query params `status` e `tipo`
- [x] `list_tasks` no serviço filtra por `status` (case-insensitive) e `tipo_relatorio`
- [x] `RelatorioHistory` exibe dropdowns de filtro: status e tipo
- [x] Auto-refresh a cada 15s quando há tarefas com status PENDING ou PROCESSING
- [x] Indicador visual "Auto-atualizando…" quando auto-refresh está ativo
- [x] Coluna ID abreviado (primeiros 8 chars) adicionada à tabela
- [x] `relatorioApi.getHistorico` aceita parâmetros opcionais `status` e `tipo`
- [x] ruff 0 erros novos introduzidos
