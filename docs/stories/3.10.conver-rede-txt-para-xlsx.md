## Story

**As a** operador do Financial P,
**I want** uma aba "Conver" com sub-aba "Rede" onde faço upload de um ou mais arquivos TXT de extrato da credenciadora Rede e recebo um XLSX estruturado para conciliação,
**so that** eu elimine o processo manual de conversão e tenha os dados prontos para análise diretamente no Excel.

## Contexto Técnico

### Formato do Input (TXT Rede)
- Encoding: `latin-1`
- Formato posicional (largura fixa), sem delimitadores
- 4 seções identificadas por cabeçalhos textuais:
  - `MOVIMENTO DE VENDAS` → crédito e débito
  - `MOVIMENTO FINANCEIRO` → pagamentos/créditos em conta
  - Seção de tarifas/débitos (identificada por cabeçalho de coluna `DATA DA INCLUSÃO`)
- Campos do cabeçalho: Estabelecimento (Nº EC), Período, Data de Emissão
- Variações de layout entre versões/estabelecimentos → fallback para Gemini Flash

### Estrutura do Output (XLSX)
4 abas com formatação profissional (cabeçalho azul escuro, linha congelada, valores em R$):

| Aba | Colunas |
|-----|---------|
| `Vendas_Credito` | Estabelecimento, Origem_Arquivo, Data_Venda, Data_Recebimento, Resumo_Vendas, Bandeira, Quantidade, Modalidade, Valor_Bruto, Valor_Correcao, Valor_Liquido, Tipo_Lancamento |
| `Vendas_Debito` | Estabelecimento, Origem_Arquivo, Data_Venda, Data_Recebimento, Resumo_Vendas, Bandeira, Quantidade, Modalidade, Valor_Bruto, Valor_Saque, Valor_Liquido, Banco_Agencia_Conta |
| `Pagamentos` | Estabelecimento, Origem_Arquivo, Data_Recebimento, Ordem_Credito, Valor_Liquido, Banco_Agencia_Conta |
| `Tarifas_e_Debitos` | Estabelecimento, Origem_Arquivo, Data_Inclusao, Data_Pagamento, Motivo_Debito, Resumo, Valor_Devido, Valor_Debitado, Meio_Pagamento |

### Arquitetura do Parser (dois estágios)
```
TXT → [1] Parser Posicional (Python/regex, primário)
              ↓ linha não reconhecida / seção inesperada
         [2] Gemini Flash (fallback, só trechos problemáticos)
              ↓
         Dados estruturados → openpyxl → XLSX
```

### Gemini Flash
- Model: `gemini-1.5-flash` (ou `gemini-2.0-flash` se disponível)
- Usado apenas quando o parser posicional retorna `None` para uma linha/seção
- Prompt: estruturado, envia o trecho problemático + contexto da seção atual
- Chave: variável de ambiente `GEMINI_API_KEY` em `apps/api/.env`
- Lib: `google-generativeai` (adicionar ao `requirements.txt` / `pyproject.toml`)

### Navegação (Sidebar)

- Módulo: `conver`, `modulePrefix: '/conver'`

## Acceptance Criteria

### AC1 — Endpoint de conversão
- `POST /api/v1/conver/rede` aceita `multipart/form-data` com campo `files` (1 a N arquivos `.txt`)
- Processa cada arquivo com o parser posicional
- Linhas não reconhecidas são enviadas ao Gemini Flash (máximo 1 chamada por arquivo)
- Retorna XLSX como `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- Nome do arquivo retornado: `Conciliacao_Rede_{periodo}.xlsx` (período extraído do TXT) ou `Conciliacao_Rede_Consolidado.xlsx` se múltiplos arquivos
- Status 400 se nenhum arquivo enviado; 422 se arquivo não for TXT válido da Rede

### AC2 — Parser posicional
- Extrai corretamente: Estabelecimento, Período, Data de Emissão do cabeçalho
- Seção `Vendas_Credito`: todas as linhas de detalhe com os 11 campos mapeados
- Seção `Vendas_Debito`: todas as linhas com os 12 campos mapeados
- Seção `Pagamentos`: todas as ordens de crédito com os 6 campos mapeados
- Seção `Tarifas_e_Debitos`: todos os débitos com os 9 campos mapeados
- Parcelas parceladas (ex: `PARC.ESTAB. 01/03`) geram linha individual por parcela

### AC3 — Fallback Gemini Flash
- `GEMINI_API_KEY` ausente → fallback desabilitado, linha problemática ignorada com log `WARNING`
- `GEMINI_API_KEY` presente → envia trecho + prompt estruturado, tenta parsear resposta JSON
- Falha no Gemini → linha ignorada com log `ERROR`, processamento continua
- Máximo de 20 linhas por chamada ao Gemini para controlar custo

### AC4 — XLSX gerado
- 4 abas conforme especificado no Contexto Técnico
- Cabeçalho: fundo `#1e3a5f`, texto branco, negrito
- Linha 1 congelada (freeze_panes)
- Colunas de valor formatadas como `R$ #.##0,00`
- Colunas de data formatadas como `DD/MM/YYYY`
- Largura das colunas auto-ajustada (mínimo 12, máximo 40 caracteres)
- Coluna `Origem_Arquivo` contém o nome do arquivo TXT de origem

### AC5 — Interface Conver/Rede
- Dropzone aceita múltiplos arquivos `.txt` via drag-and-drop ou clique
- Lista os arquivos selecionados com nome e tamanho
- Botão "Converter" ativo apenas quando há pelo menos 1 arquivo
- Durante processamento: spinner + mensagem "Processando X arquivo(s)..."
- Sucesso: botão "Baixar XLSX" aparece automaticamente com download direto
- Erro: mensagem de erro legível (ex: "Arquivo inválido: não é um extrato Rede")
- Sem autenticação adicional — qualquer usuário logado acessa

### AC6 — Sidebar e navegação
- Seção "Conver" aparece no Sidebar entre "Relatórios" e "Configurações"
- Sub-item "Rede" com ícone `FileOutput` e href `/conver/rede`
- Highlight ativo correto ao navegar para `/conver/rede`
- Módulo verificado via `podeAcessar('conver')` — fallback permissivo (retorna `true` se módulo não configurado)

### AC7 — Testes unitários
- `test_rede_parser.py`: testa parser com arquivo real (fixture do TXT de janeiro/2021)
  - Testa extração do cabeçalho
  - Testa linhas de crédito à vista e parceladas
  - Testa linhas de débito
  - Testa pagamentos
  - Testa tarifas
- `test_conver_endpoint.py`: testa endpoint com mock do XLSX retornado
- Cobertura mínima: 80% do `rede_parser.py`
