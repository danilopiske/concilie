# Story 3.17: Resumo Executivo por Cliente

## Status
Done

## Objetivo
Adicionar um painel de resumo executivo na página de detalhes do cliente (`/gestao/clientes/[id]`) mostrando: importações recentes, cálculos recentes, relatórios recentes e KPIs de notificações — tudo numa única visão contextual por cliente.

## Dependências
- Story 3.15 (notificações in-app) — modelo Notificacao
- Story 3.16 (centro de tarefas) — modelos de tasks

## Acceptance Criteria
- [x] Endpoint `GET /api/v1/clientes/{cliente_id}/resumo` retorna dados consolidados
- [x] ImportTask filtradas por `cliente_id` (relação direta)
- [x] CalculoTask e RelatorioTask: 5 mais recentes globais (sem FK para cliente)
- [x] Notificações não lidas do usuário atual (global + do usuário)
- [x] Frontend: `ClienteResumoPanel` com KPIs e listas de tasks recentes
- [x] Página `/gestao/clientes/[id]/page.tsx` criada com painel integrado
- [x] ruff check: 0 erros

## Arquitetura

### Backend — Modelos usados

| Modelo | Tabela | Filtro |
|--------|--------|--------|
| Cliente | clientes | `cliente_id` (PK) |
| ImportTask | import_tasks | `cliente_id` (FK direto) |
| CalculoTask | calculo_tasks | sem FK cliente — global top-5 |
| RelatorioTask | relatorio_tasks | sem FK cliente — global top-5 |
| Notificacao | notificacoes | `usuario_id` == current_user ou None |

### Response Schema
```json
{
  "cliente_id": 1,
  "nome": "Cliente XYZ",
  "cnpj": "00.000.000/0001-00",
  "notificacoes_nao_lidas": 3,
  "import_tasks_recentes": [{"id": "...", "status": "SUCCESS", "created_at": "..."}],
  "calculo_tasks_recentes": [{"id": "...", "status": "SUCCESS", "created_at": "..."}],
  "relatorio_tasks_recentes": [{"id": "...", "status": "SUCCESS", "progress": 100, "created_at": "..."}]
}
```
